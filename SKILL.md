# Mandrama Export Adaptation

This skill is a **market-aware adaptation system**, not a translation tool. It converts China-first scripted material into overseas-ready project packages optimized for audience fit, monetization, and production feasibility.

## When to use

- Adapt a Chinese web drama / manhua / short drama for overseas release
- Rewrite a domestic plot for English-speaking or region-specific audiences
- Package a project for YouTube / international vertical drama / streaming clips
- Decide whether a project should be Anglophone-led, region-first localized, subtitle-first, or multilingual
- Choose whether a title should be 2D anime, motion comic, or 3D MetaHuman photoreal

## Core objective

Maximize:
- International watch-through rate
- Click-through and hook rate
- Monetizable audience quality
- Sponsorability and premium brand compatibility
- Adaptation efficiency per unit of production cost

Subject to:
- Preserving the original emotional engine
- Not breaking core plot logic
- Keeping production feasible in an AI-native pipeline
- Avoiding culture-locked exposition that collapses outside the target context

## Non-goals

- Do NOT produce word-for-word translation unless explicitly asked
- Do NOT preserve culturally opaque references just because they exist in the source
- Do NOT default to any one export language as a universal answer
- Do NOT assume "foreign audience = likes everything Japanese"
- Do NOT force 3D photoreal on all genres
- Do NOT confuse "good script" with "good export packaging"

## Default strategic posture

Unless the user overrides:
- **Primary export language should match the target market and monetization plan**
- **If market info is missing, start by testing an Anglophone export hypothesis** because the current operator priority is US/UK/Europe-facing growth
- **Japanese is one option, not the only non-English path**
- **Spanish / Portuguese / Arabic / French / German / Korean and other regional packages are valid when market logic supports them**
- **Chinese is source language, not automatically the export headline language**
- **Adaptation > translation**
- **Retention and monetization > fidelity to surface details**

## Workflow

Follow these 6 phases in order:

### Phase 1 - Source diagnosis
Classify the source material using [`references/decision-framework.md`](references/decision-framework.md):
- Identify the emotional engine
- Map export friction points
- Find universal hooks
- Score format fit

### Phase 2 - Route selection
Choose the market and language strategy using [`references/market-strategy-rules.md`](references/market-strategy-rules.md):
- Anglophone premium export
- Performance-led subculture export
- Region-first localized launch
- Multilingual layered release

### Phase 3 - Narrative adaptation
Rewrite the project using [`references/adaptation-layers.md`](references/adaptation-layers.md):
- One-line sell, character archetypes, social logic, dialogue, episode architecture, packaging
- Consult [`references/archetype-glossary.md`](references/archetype-glossary.md) for character role mappings

### Phase 4 - Dialogue localization
Localize dialogue using [`references/dialogue-localization-engine.md`](references/dialogue-localization-engine.md):
- Localize, do not translate
- Provide tone variants and dub/subtitle modes

### Phase 5 - Format and packaging
Select production format and packaging using the format rules in [`references/market-strategy-rules.md`](references/market-strategy-rules.md).

### Phase 6 - Final synthesis
Choose output depth based on the request:
- For narrow route / language / format questions, use [`templates/quick-decision.md`](templates/quick-decision.md)
- For full adaptation / packaging requests, use [`templates/output-package.md`](templates/output-package.md)

Use individual templates as needed:
- [`templates/adaptation-memo.md`](templates/adaptation-memo.md)
- [`templates/risk-register.md`](templates/risk-register.md)
- [`templates/packaging-sheet.md`](templates/packaging-sheet.md)
- [`templates/assumption-table.md`](templates/assumption-table.md)

## Validation

Before finalizing output:
1. Check all results against [`references/red-lines.md`](references/red-lines.md) - flag and correct any violations
2. Score results using [`references/rubrics.md`](references/rubrics.md) - any dimension scoring <=2 triggers a rewrite

## Response style

- Speak as a **story strategist + export producer**, not a translator
- Explicitly separate: **Fact**, **Inference**, **Assumption**, **Recommendation**
- Prefer decisive ranking over "it depends"
- Prefer trade-off clarity over vague praise
- Prefer examples over abstraction
- Do not recommend all options equally - pick one and justify
